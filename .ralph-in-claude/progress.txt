- reg_addr was already 8-bit in original design

- Learnings: 8-bit arithmetic overflow handles wrap-around naturally

## 2026-03-07T21:13+0800 - US-003: I2CDriver 基礎框架
- Created backend/sim/i2c_driver.py with I2CDriver class: __init__, reset(), delay()
- Files changed: backend/sim/i2c_driver.py
- Learnings: Wrapper uses slave_addr_in (not slave_addr) for master target address. Reset pattern: 5-cycle assert, 5-cycle settle.

## 2026-03-07T21:15+0800 - US-004: I2CDriver Write 操作
- Added write_bytes() to I2CDriver. Splits into chunks of 14 data bytes (num_bytes is 4-bit, max 15 minus 1 reg pointer). Monitors byte_count to feed data_in. Returns ACK/NACK status.
- Files changed: backend/sim/i2c_driver.py
- Learnings: num_bytes is 4-bit (max 15); master samples data_in during ACK HIGH_MID tick; done pulse is 1 cycle wide
---

## 2026-03-07T21:17+0800 - US-005: I2CDriver Read 操作
- Added read_bytes() to I2CDriver. Uses stop-start sequence (write reg pointer, then read). Chunks at 15 bytes per read transaction.
- Files changed: backend/sim/i2c_driver.py
- Learnings: Max read per transaction is 15 (vs 14 for write since no reg pointer byte). Check done before data_valid to avoid infinite loop.
---

## 2026-03-07T21:19+0800 - US-006: I2CDriver Scan 與 Register Dump
- Added scan() and get_register_dump() to I2CDriver. scan sends 1-byte write to check ACK. get_register_dump reads all 256 registers via DUT hierarchy.
- Files changed: backend/sim/i2c_driver.py
- Learnings: DUT hierarchy is dut.dut.slave_inst.register_file[i] (wrapper names i2c_top as "dut", slave as "slave_inst")
---
